# Runtime 0.0 Finalization — Implementation Plan

Companion to `../specs/2026-07-12-deepseek-application-agent-runtime-0.0-design.md`.
This plan turns the §11 verification matrix and §13 release gate into concrete
work, ordered so the foundational test infrastructure lands first.

## Key finding from code investigation

The DeepSeek / Chat Completions supported path is **wire-agnostic by
construction**:

- Auto-compaction trigger (`core/src/session/context_window.rs`,
  `session/turn.rs`), the limit (`protocol/src/openai_models.rs:441`), dispatch
  (`turn.rs:916`), and the local executor (`compact.rs` → `stream_chat_completions`)
  have no Responses-only branch. `supports_remote_compaction()` is hard-coded
  `false`, so DeepSeek always summarizes via an ordinary chat/completions call —
  no `/responses/compact` dependency.
- Persist/resume stores and replays internal `ResponseItem`s
  (`RolloutItem::ResponseItem`); `Reasoning.content` (DeepSeek `reasoning_content`)
  and `FunctionCall.call_id`/`arguments` round-trip unchanged. No
  Chat-Completions branching in the recorder or reconstruction.
- Context accounting is fed by provider-reported usage parsed in
  `codex-api/src/chat_completions/sse_parser.rs`.

**But**: every end-to-end integration test mocks only `wire_api = "responses"`
(60 occurrences in app-server tests, 0 for chat_completions). So the path works
but is unverified end-to-end. The keystone of this plan is a reusable mock
chat_completions SSE server; everything else builds on it.

## Constraints

- Do not change `ResponseEvent`, the app-server protocol, or prompt/tool/history
  semantics. Offline tests only; no network in CI.
- Reuse existing wire-agnostic machinery (compaction, resume, app-server
  lifecycle). Add DeepSeek-path *coverage*, not parallel implementations.
- Verify with nextest (`RUST_MIN_STACK=8388608`), provider env vars unset.
- No new external dependencies unless unavoidable.

## Tasks

### Phase A — Foundation

- [ ] **A1: Mock chat_completions SSE server.** New `core/tests/common/chat_completions.rs`
  (registered in `core/tests/common/lib.rs`), mirroring `responses.rs`:
  `cc_sse(chunks)` emitting bare `data: {json}\n\n` + `data: [DONE]`, chunk
  fixtures (`cc_role_content`, `cc_content_delta`, `cc_reasoning`,
  `cc_tool_call_open`, `cc_tool_call_args`, `cc_finish`, `cc_usage`),
  `mount_chat_completions_once/_sequence` matching `.*/chat/completions$`,
  `start_mock_chat_completions_server`. App-server wrapper in
  `app-server/tests/common/mock_model_server.rs` +
  `write_mock_chat_completions_config_toml` in `config.rs` (`wire_api =
  "chat_completions"`, `env_key`). Include a self-test that drives one turn.

### Phase B — Offline gate on the DeepSeek path

- [ ] **B1: Thinking-effort mapping unit test.** Assert `deepseek_thinking_for_effort`
  matches the §4.2 table (none/minimal/low→disabled, medium→enabled,
  high/max→annotated 8192, xhigh/ultra→annotated 16384) and its serialized JSON
  (`type: disabled|enabled|annotated`). Cheap, no harness.
- [ ] **B2: SSE parser gap tests.** reasoning+tool-call interleave; multiple tool
  calls (2+ indices) in one response; malformed/partial SSE chunk skipped;
  transport error → `ApiError::Stream`; idle-timeout error.
- [ ] **B3: End-to-end chat turn tests (mock).** text; reasoning+text; single tool
  call → result → continuation; sequential tool loop; cancellation mid-stream.
- [ ] **B4: Auto-compact boundary on chat path (mock).** Scaled-down limit; assert
  compaction triggers at/over threshold and the turn completes (compact then
  continue). Mirror `compact.rs` Responses tests but with the chat mock.
- [ ] **B5: Persist→resume→continue on chat path (mock).** Two turns incl. one tool
  call, persist/stop, resume, continue; assert reasoning_content + call_id
  survive into the next chat request and no duplicate tool side effect.
- [ ] **B6: App-server lifecycle smoke on chat provider (mock).** initialize/
  initialized, thread start, turn start→stream→completion, cancellation over a
  `wire_api = "chat_completions"` provider.
